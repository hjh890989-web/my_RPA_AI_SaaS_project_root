# HITL (Human-in-the-Loop)

## 📖 개념 정의
**HITL(Human-in-the-Loop)**은 AI 시스템의 판단과 의사결정 과정에 반드시 인간 개입을 필수로 거치게 만드는 안전 프로토콜입니다. 제조 공정에서 AI가 단독으로 오판하여 시스템 정지, 불량 승인 등의 중대 사고를 내는 것을 방지하기 위해 설계되었습니다.

## 🛡️ FactoryAI의 HITL 4대 원칙
1. **AI 제안, 인간 확정**: AI가 생성한 어떠한 결과물(로깅, 이상 탐지 등)도 데이터베이스상 `PENDING` 상태로 저장되며, 외부로 발행되거나 다음 공정으로 넘어가기 위해선 반드시 관리자의 Approve(승인)를 받아야 합니다. 승인 없이는 자동 발행이 원천 차단됩니다.
2. **판단 근거 의무 표시 (XAI)**: AI가 어떤 판단(예: 불량 탐지)을 내렸을 때 "왜 그런 결론이 나왔는지"를 인간이 읽을 수 있는 한국어 설명으로 의무 표시해야 합니다.
3. **단독 실행 금지**: AI가 공장 설비의 전원을 끄거나(STOP), 생산 일정을 강제 변경(CHANGE)하는 물리적 연산 제어를 단독으로 실행하지 못하도록 API 게이트웨이 레벨에서 하드코딩 차단됩니다.
4. **롤백/수정 보장**: AI가 잘못 인식한 값은 관리자가 언제든 원클릭으로 Reject/수정할 수 있어야 하며, 원본 데이터(음성, 사진)는 무결성 검증을 위해 반드시 보존되어야 합니다.
