## 1. 제품 개요 (Product Overview)

### 1-1. 제품 비전 (Product Vision)
**"중소 제조사에 1도의 수기 입력 없이, 원청사의 혹독한 규제/감사로부터 생존할 수 있는 무결점 데이터를 제공한다."**

### 1-2. 타겟 시장 및 사용자 (Target Audience)
*   **Target Market**: ERP/MES(스마트공장 기초)를 도입했으나 현장 활용에 실패한 매출 100억~500억 규모 제조 부품사 (SEG-C)
*   **주요 사용자군**: 
    1.  **결재자(구매자)**: 원청사 감사에 생존권이 걸린 구매/품질본부장, 자부담을 극도로 꺼리는 CFO
    2.  **실무자(운영자)**: 현장의 기존 수기 방식/키오스크 조작에 저항하는 50대 반장 및 작업자

### 1-3. 가치 제안 (Value Proposition)
효율 10% 증가가 아닌, **비용(자부담) 0원 + 수기 입력 0번 + 야근 0일(원클릭 감사대응)** 이라는 확실한 진통제 제공.

## 2. 성공 지표 (Success Metrics)

이 MVP가 성공적인 Product-Market Fit(PMF)에 도달했는지 확인하기 위한 측정 지표입니다.

| 지표 유형 | 항목명 | 목표치 (MVP 기준) | 설명 |
| :--- | :--- | :--- | :--- |
| **North Star** | **Manual Input Rate (수기 입력률)** | **0% 근접** | 현장 작업자가 키보드/터치를 통해 공정 데이터를 입력하는 횟수 (핵심 저항 지표) |
| Core KPI 1 | **Time to Report (리포트 생성 시간)** | **10분 이내** | 원청사 실사용 Traceability(제조 이력) 추적 리포트를 출력하는 데 걸리는 시간 (기존: 수일 야근) |
| Core KPI 2 | **POC Conversion Rate (시연 후 계약 전환율)** | **30% 이상** | CFO 대상 '바우처 대행+자부담 시뮬레이션' 후 실제 계약이 이뤄지는 비율 |

## 3. 핵심 에픽 및 사용자 스토리 (MVP Scope)

개발을 위해 기능을 4대 Epic(대분류)으로 나누고, 사용자 스토리(User Story) 형태로 구현 스펙을 정의합니다.

### 🟡 E1. 무입력 패시브 로깅 (Passive Data Sensing)
> 작업자의 저항(행동 변화 요구)을 없애기 위한 현장 데이터 자동 수집 인프라

*   **US-1.1**: 현장 작업자로서, 나는 기계 패널이나 공정의 완성 사진 한 장만 스마트폰으로 찍으면, AI가 공정 상태/설비명을 자동으로 읽어내어 기록하게 하고 싶다. (Vision OCR/AI)
*   **US-1.2**: 현장 작업자로서, 나는 장갑을 벗지 않고 "2번 라인 금형 교체 끝"이라고 말만 하면, 음성 인식(STT)+LLM이 텍스트로 변환해 공정 일지에 정형화된 데이터로 입력하게 하고 싶다.
*   **Acceptance Criteria (완료 기준)**: 
    * 키오스크나 키보드 없이 앱 실행 후 버튼 1회 또는 음성만으로 작업 지시가 완료되어야 함.
    * 노이즈가 심한 공장 환경에서 음성 인식률 85% 이상 확보 (또는 오터치 시 1클릭 복구 UI 제공).

### 🔵 E2. 레거시(ERP/MES) API 브릿지 모듈
> 비싼 신규 구축 없이, 사일로화된 기존 시스템에서 생명줄 데이터를 가져오는 백엔드

*   **US-2.1**: IT 담당자/CIO로서, 나는 솔루션이 우리 회사의 기존 더존/영림원 ERP DB에서 기초 데이터를 API로만 긁어와 원래 시스템의 보안이나 코드를 절대 망치지 않았으면 좋겠다.
*   **US-2.2**: 경영진으로서, 나는 새로 도입한 AI 시스템 데이터와 기존 ERP 재고 데이터가 실시간으로 일치하는 것을 하나의 화면에서 보고 싶다.
*   **Acceptance Criteria**:
    * Read-Only 커넥터를 통해 기존 DB 마이그레이션 책임을 최소화하는 아키텍처.
    * 국내 Top 3 ERP 연동 규격을 모듈화하여 1주일 내 데이터 연동 보장.

### 🔴 E3. 원클릭 감사 리포터 (Traceability Report Gen)
> 실질적인 구매 트리거. 복잡한 데이터를 결합해 '규제 방어막'을 생성하는 기능

*   **US-3.1**: 품질이사/구매본부장으로서, 나는 원청사 감사팀이 떴을 때 특정 로트(Lot) 번호를 검색하면, 해당 제품의 생산 이력과 투입 자재가 한눈에 보이는 PDF 리포트를 10초 만에 다운로드 받고 싶다.
*   **US-3.2**: 품질 실무자로서, 나는 원청사마다 요구하는 엑셀/PDF 양식이 다른데, 이것에 맞춰 드래그 앤 드롭으로 리포트 출력 포맷을 쉽게 세팅해두고 싶다.
*   **Acceptance Criteria**:
    * 데이터 누락 시(수집 실패 구간) 강제로 알림을 주는 결함 체커 기능 필수 포함 (데이터 조작 의심 원천 차단).
    * 다운로드 된 PDF 문서는 삼성/현대차 등 주요 글로벌 벤더사 표준 데이터 필드를 충족해야 함.

### 🟢 E4. 진단 및 결재 무기화 (B2B Sales Enabler)
> CFO의 승인 딜레이를 무력화하는 영업용 솔루션 (제품 안에 내장)

*   **US-4.1**: 영업담당자(당사)로서, 나는 잠재 고객사 미팅 시 현재 스펙을 입력하면 정부 바우처 몇 프로(%)를 타서 자부담이 얼마 나오는지 실시간 웹 브로셔 시뮬레이터를 보여주고 싶다.
*   **US-4.2**: CFO로서, 나는 영업팀이 보여준 1페이지 보고서 안에 예상 절감 시간(야근 비용 삭감분)과 내가 내야 할 실제 초기 비용이 간결히 적혀 있어, 대표에게 올릴 품의서 작성 시간을 없애고 싶다.
*   **Acceptance Criteria**:
    * "10억짜리 솔루션 ➔ 바우처 80% 지원 ➔ 귀사 부담 2,000만 원" 구조를 즉각 시각화하는 간단한 웹 페이지.

## 4. 로드맵에서 제외된 사항 (Out of Scope for MVP)

초기 PMF(Product-Market Fit) 검증의 속도를 위해 **MVP 단계에서는 절대 개발하지 않고 Phase 2 로드맵으로 미루는 항목**입니다.

1.  **AI 최적화 스케줄러 (강력하지만 무거움)**: COO의 퇴사 리스크 해소에는 궁극적으로 필요하나, 최적화 알고리즘 개발 난이도(Rank 5)가 너무 높고 현장 변수가 많아 초기 실패율이 높습니다. MVP는 '빈틈없는 기록(Logging)'에 집중합니다.
2.  **완전 수축형 온프레미스(On-premise) 보안 인프라**: 망분리가 절대적인 일부 기업(방산, 핵심 부품)은 타겟에서 잠시 제외하고, 클라우드/API 연동에 우호적이거나 보안 협의가 가능한 Tier-2 기업들을 먼저 공략합니다.
3.  **XAI (설명가능한 AI 분석창)**: 센서 이상 탐지를 위한 복잡한 인과관계 분석 모델은 데이터가 최소 3~6개월 축적된 이후에 의미가 있습니다.
