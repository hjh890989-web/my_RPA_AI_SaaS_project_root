## 1. 핵심 요약

### 실패 요인 우선순위 매트릭스

| 순위 | 실패 요인 | 국내 양적 근거 | 글로벌 양적 근거 | 도입 전 vs. 도입 후 |
|------|----------|-------------|----------------|-------------------|
| **1** | **ROI 불확실성 / 투자 실패 공포** | AI 전환 효과 부정 60%+ | DX 프로젝트 실패율 70% (McKinsey) | **도입 前** (가장 강한 저항) |
| **2** | **정보 비대칭 / 검증 도구 부재** | 업종별 성공사례 부재, PoC 함정 | RPA 초기 프로젝트 30~50% 실패 | 도입 前·中 |

## 2. 실패 요인 ① 투자 실패 리스크 공포 (ROI 불확실성)

### 2-1. 정의

중소기업 대표가 AI/RPA 도입을 검토할 때 가장 먼저 부딪히는 심리적 장벽. "효과가 있을까?"의 불확신이 구매 결정을 가로막는다.

### 2-2. 양적 근거

**[국내 데이터]**

| 지표 | 수치 | 출처 |
|------|------|------|
| "AI 전환 효과가 미미할 것" 부정 응답 | **60% 이상** | 동아일보·EY한영 설문 |
| AI 도입 비용 부담 응답 | **70~80%** | 대한상공회의소·중소기업중앙회 복수 조사 |
| AI 전문 인력 미보유 응답 | **80% 이상** | 동일 설문 |
| 스마트공장 도입 후 '업무 개선 효과 없음' 응답 | **15%** | 중소벤처기업부 실태조사 |
| 도입 후 '시스템 결함·불안정' 불만 | **33%** | 동 실태조사 |

**[글로벌 데이터]**

| 지표 | 수치 | 출처 |
|------|------|------|
| 디지털 전환 프로젝트 목표 미달률 | **약 70%** | McKinsey Global Institute |
| RPA 초기 프로젝트 실패·확장 불가 비율 | **30~50%** | Forrester / 업계 공통 추정치 |
| AI 성숙도 최고 단계 도달 기업 비율 | **약 1%** | McKinsey |
| RPA 도입 후 과도한 기대→조기 실망 패턴 비율 | 대다수 | Gartner Hype Cycle 분석 |

### 2-3. 공포의 구체적 형태

```
① 매몰비용 공포: "몇천만 원 들였다가 안 되면 어떡하나"
② 기회비용 공포: "그 돈에 직원 한 명 더 뽑는 게 낫지 않나"  
③ 경쟁비용 공포: "남들은 다 하는데 나만 안 하면 뒤처지는 건 아닌가"
④ 책임 공포: "내 결정으로 회사 돈 날리면 어떡하나" (중간 관리자)
```

### 2-4. 이 공포가 특히 강한 이유: 중소기업 고유 구조

- **의사결정자 = 대표 본인**: 실패의 책임이 100% 자신에게 귀속 → 리스크 회피 성향 극도로 강함
- **레퍼런스 의존**: "우리 같은 회사가 해봤나?"가 첫 번째 구매 필터 → 없으면 STOP
- **회수 기간 압박**: 연간 IT 예산 자체가 수백~수천만 원 수준 → 실패 1회 = 연간 예산 소진

## 3. 실패 요인 ② 정보 비대칭 — 검증 도구의 부재

### 3-1. 정의

AI/RPA 공급자(기술 우위)와 수요자(도메인 우위)의 정보 격차로 인해 **수요 기업이 솔루션의 적합성을 스스로 판단할 방법이 없는 상태**. 이것이 공포를 증폭시키고 구매 결정을 마비시킨다.

### 3-2. 양적 근거

**[국내 데이터]**

| 지표 | 수치 | 출처 |
|------|------|------|
| "업종 특화 성공 사례를 찾기 어렵다" 응답 | 다수 응답 | 중소기업 AI 도입 실태 분석 (ZDNet·매일경제) |
| 스마트공장 도입 기업 중 'AI 기반 고도화' 달성 | **0.1%** | 중기부 스마트제조혁신 실태조사 (2025년 발표) |
| 도입 기업의 75.5%가 기초 수준 정체 | **75.5%** | 동 실태조사 — "데이터는 쌓이는데 활용 못 함" |
| PoC 성공 후 실제 현장 적용 실패 (정성 사례) | 반복적으로 등장 | 업계 전문가 분석, AI Times |

**[글로벌 데이터]**

| 지표 | 수치 | 출처 |
|------|------|------|
| RPA 실패 원인 중 "잘못된 사용처 선택" | **40% 예외 이상 포함 프로세스** 자동화 시 대부분 실패 | Naviant / Forrester 분석 |
| PoC는 성공했으나 실제 비즈니스 적용 실패 패턴 | 매우 빈번 | CIO.com / 업계 공통 분석 |
| "기술 가능성만 검증하고 비즈니스 가치 미검증" PoC | 대다수 | 업계 전문 보고서 |

### 3-3. 정보 비대칭의 3가지 구체적 발현

**① 사전 검증 도구 부재**
```
고객이 알고 싶은 것                      현실
─────────────────────────────────────
"우리 업무에 적용 가능한가?" ─────→ "해봐야 압니다 (PoC 필요)"
"절감 효과가 얼마나 되나?"  ─────→ "도입해보면 압니다"
"몇 개월 만에 ROI 나오나?" ─────→ "고객사마다 달라요"
```
→ **결론: 사기당할지 안 당할지 알 방법이 없다** → 도입 포기

**② PoC(개념검증)의 구조적 함정**
- PoC는 통제된 환경에서 기술 작동 여부만 확인
- 실제 현장의 예외상황(주문 변경, 시스템 다운, 담당자 교체)은 PoC에 반영 안 됨
- Forrester 분석: 프로세스 내 **예외 케이스가 40% 이상**이면 RPA는 실패 확률 급등
- 중소 제조현장의 실제 프로세스는 예외가 40% 이상인 경우가 대부분

**③ 공급자-수요자 간 언어 격차**
```
공급자 사용 언어          수요자가 이해하는 언어
─────────────────────────────────────
LLM, RAG, 오케스트레이션  → "그게 뭔데요?"
자동화봇, API 연동       → "우리 더존이랑 되나요?"
ROI, CAGR, SLA         → "한 달에 얼마 아끼나요?"
```
→ 공급자가 수요자 언어로 말하지 못하면, 수요자는 신뢰하지 않는다.

## 4. 실패 요인 ③ 초기 비용 부담

### 4-1. 양적 근거

| 지표 | 수치 | 출처 |
|------|------|------|
| AI 도입 장벽 1위: 초기 비용 | **44.2%** | 중소벤처기업부 실태조사 |
| AI 전환 비용 "부담된다" 응답 | **70~80%** | 대한상공회의소·복수 설문 |
| 스마트공장 평균 도입 비용 | **7억 5,000만 원** | 중기부 실태조사 |
| 중소기업 연간 IT 예산 (평균) | **500만~3,000만 원** | Porter 분석 기반 추정 |

### 4-2. 왜 ①, ②보다 순위가 낮은가

> **비용은 해결책이 있다. 공포와 정보 비대칭은 해결책이 없다.**

- 비용 장벽 → 정부 바우처(최대 2억 원), 성과 기반 계약(Success Fee)으로 우회 가능
- 투자 공포 → 바우처가 있어도, "효과 있을지 모르겠다"면 신청도 안 함
- 즉, **비용 장벽은 공포와 정보 비대칭이 해소된 후에야 의미 있는 변수가 된다**

**실무 근거**: 정부 바우처로 고객 부담 0원에 가깝게 해도 도입 포기하는 기업이 상당수 존재
→ 이는 "비용 문제가 아니라 신뢰·검증 문제"임을 방증

## 5. 실패 요인 ④ 전문 인력 부족

### 5-1. 양적 근거

| 지표 | 수치 | 출처 |
|------|------|------|
| "AI 활용 전문 인력 없다" 응답 | **80% 이상** | 복수 설문 합산 |
| AI 도입 장벽 2위: 인력 부족 | **20.5%** | 중소벤처기업부 실태조사 |
| AI 도입 장벽 3위: 전략 부재 | **14.9%** | 동 실태조사 |
| AI 엔지니어 연봉 (상위권) | **1억 원 이상** | 업계 추정 |

### 5-2. 왜 ①, ②보다 순위가 낮은가

- 전문 인력 부족 → 외주(SI/컨설팅)로 해결 가능
- 장기 유지보수 단계에서는 중요하지만, **도입 결정 단계에서는 공포/정보 비대칭이 더 먼저 작동**
- RPA 실패 분석: "외부 컨설턴트 의존 후 내재화 실패"는 **도입 이후 문제**이지, 도입 전 의사결정을 막는 첫 번째 요인이 아님

## 6. 실패 요인 ⑤ 프로세스·데이터 준비 미흡

### 6-1. 양적 근거

| 지표 | 수치 | 출처 |
|------|------|------|
| 도입 후 '시스템 결함·불안정' 1위 불만족 | **33%** | 중기부 실태조사 |
| 도입 후 '업무개선 효과 없음' | **15%** | 동 실태조사 |
| 도입 후 '유지보수 어려움' | **12%** | 동 실태조사 |
| 스마트공장 도입 기업 중 기초 수준 정체 | **75.5%** | 동 실태조사 |
| RPA 실패: 비표준화 프로세스 자동화 | 업계 공통 Top 3 원인 | Naviant, CIO.com |
| RPA 실패: Forrester "Rule of Five" | 의사결정 5개 초과 / 앱 5개 초과 프로세스 | Forrester Craig LeClair |

### 6-2. 왜 ①, ②보다 순위가 낮은가

- 이 문제는 **도입 이후**에 발생하는 문제
- 그러나 이 실패 경험이 미래 ①번 공포를 강화하는 **피드백 루프** 형성
- 즉, ⑤의 실패 → ①의 공포 증폭 → 더 많은 기업이 도입 포기하는 악순환
- 따라서 비즈니스 개입 지점: ⑤를 방지하는 것이 ①을 해소하는 가장 강력한 방법

## 7. 실패 요인 ⑥ 내부 변화 저항 (조직·문화)

### 7-1. 양적 근거

| 지표 | 수치 | 출처 |
|------|------|------|
| DX 실패 원인 중 '인적 저항·문화' | **1위** | McKinsey 분석 |
| 경영진 톱다운 도입 후 현장 수동 회귀 사례 | 빈번 | pointdaily.co.kr, kitim.org |
| '직원들이 사용 못 할 것' 구매 저항 | 중소 제조 대표 설문 상위 응답 | Porter 분석 자료 기반 |
| 현장 직원 참여 없는 스마트공장 도입 → 조기 중단 | 빈번 | 스마트공장 전문 미디어 |

### 7-2. 왜 ①, ②보다 순위가 낮은가

- 중소기업 구조에서는 "대표 결정 = 실행". 대형 기업과 달리 내부 저항이 도입 '결정'을 막지 않음
- 오히려 중소기업은 내부 저항보다 **대표 본인의 ROI 공포**가 결정적
- 내부 저항은 **도입 '성공'을 막는 요인**이지, 도입 '결정'을 막는 1차 요인이 아님

## 8. 요인 간 상대적 중요도 비교 분석

### 8-1. 실패 발생 시점별 분류

```
[도입 前 단계] ─ 구매 결정을 막는 요인
  ① 투자 실패 공포 (ROI 불확실성)   ████████████ 가장 강력
  ② 정보 비대칭 / 검증 도구 부재    ███████████  2위
  ③ 초기 비용 부담                  ████████     3위 (바우처로 해소 가능)
  ④ 전문 인력 부족                  █████        (외주로 해소 가능)

[도입 中·後 단계] ─ 도입 성공을 막는 요인  
  ⑤ 프로세스/데이터 준비 미흡       ████████████ 조기 실패 핵심
  ⑥ 내부 변화 저항                  ███████      장기 안착 방해
```

### 8-2. 중요도 비교 — 주요 수치 크로스체크

| 비교 기준 | ① 투자 공포 | ③ 비용 부담 | 비율 |
|----------|-----------|-----------|------|
| 부정적 응답 비율 | 60% 이상 (변환 효과 부정) | 44.2% (비용 장벽 1위) | **투자 공포 > 비용 장벽** |
| 바우처 지원 시 해결 여부 | **해소 안 됨** | **해소 가능** | 투자 공포가 근본 장벽 |
| 도입 결정 차단 효과 | **도입 전** | 도입 전 | 동일 단계지만 공포가 선행 |

| 비교 기준 | ② 정보 비대칭 | ④ 인력 부족 | 분석 |
|----------|-------------|-----------|------|
| 해결 방법의 존재 여부 | **외부 수단으로 해소 어려움** | 외주로 해소 가능 | 정보 비대칭이 더 근본적 |
| 도입 결정 차단 효과 | **도입 전 결정 자체를 마비** | 도입 후 유지보수 단계 | 정보 비대칭이 선행 |
| RPA 글로벌 실패 데이터 | PoC 성공 후 현장 실패 빈번 | 컨설턴트 철수 후 내재화 실패 | 두 요인 모두 중요하나 시점 다름 |

### 8-3. 왜 ①② 가 핵심인가 — 인과관계 다이어그램

```
[정보 비대칭]
  ↓
"적합한 솔루션인지 판단 불가"
  ↓
[투자 실패 공포]
  ↓
"해봤다가 돈 날리면 어떡하나"
  ↓
→ 구매 결정 마비 (도입 포기)
→ 또는: 섣불리 도입 → 프로세스 미흡(⑤) → 실패 → ①의 공포 재강화
```

**결론**: ①과 ②는 다른 모든 요인의 **원인이자 증폭기**이다.  
비용(③), 인력(④), 프로세스(⑤), 문화(⑥) 문제가 해결되어도 ①②가 해소되지 않으면 도입은 일어나지 않는다.

## 9. 핵심 인사이트 및 사업적 시사점

### 9-1. 이 분석이 우리에게 말해주는 것

> **"기술을 파는 것이 아니라, 공포를 해소하고 검증 경험을 파는 것이다."**

| 실패 원인 | 우리가 줄 수 있는 해답 |
|---------|---------------------|
| ① 투자 실패 공포 | **수치화된 ROI 사전 시뮬레이션** + 성과 기반 계약 (실패 시 환불 옵션) |
| ② 정보 비대칭 | **업종별 검증 케이스 공개** + 무료 PoC + 직관적 현장 진단 도구 |
| ③ 비용 부담 | **바우처 신청 전 과정 대행** + 자부담 최소화 설계 |
| ④ 인력 부족 | **우리가 유지보수·운영까지 책임** (구축+운영 묶음 계약) |
| ⑤ 프로세스 미흡 | **도입 전 프로세스 진단** 선행 → 실패 가능성 사전 제거 |
| ⑥ 내부 저항 | 현장 담당자 교육 포함, **30일 현장 밀착 지원** |

### 9-2. '검증 가능한 직관적 도구'가 왜 게임 체인저인가

현재 시장의 가장 큰 구멍(Gap):
```
고객이 원하는 것: "도입 전에 내 공장에서 얼마나 효과가 나는지 미리 알고 싶다"
현재 가능한 것: "도입하고 3~6개월 써봐야 압니다"
```

이 갭을 메우는 도구(예시):
- **RPA 적합성 진단 체크리스트**: "이 5가지 조건 충족 시 성공률 80%" 형태
- **업무 시간 절감 계산기**: 현재 업무 시간 입력 → 예상 절감 시간·금액 자동 산출
- **동종 업계 사례 비교 카드**: "금속 가공업 N=15개사, 평균 3.2개월 만에 손익분기점 도달"

이런 도구의 존재 자체가 **정보 비대칭을 줄이고 신뢰를 만든다**.

### 9-3. 실패 요인이 우리에게 열어주는 시장 기회

```
실패 원인의 역(逆) = 시장 기회

① ROI 불확실성  →  ROI를 수치로 사전 예측·사후 증명하는 서비스
② 정보 비대칭   →  직관적 자가 진단 도구 + 업종 사례집 제공
③ 비용 부담     →  바우처 연계 All-in-one 패키지
④ 인력 부족     →  구축+운영+유지보수 묶음 서비스 (Managed Service)
⑤ 프로세스 미흡 →  도입 前 프로세스 설계 컨설팅 선행
⑥ 변화 저항     →  현장 밀착 교육·온보딩 프로그램
```
