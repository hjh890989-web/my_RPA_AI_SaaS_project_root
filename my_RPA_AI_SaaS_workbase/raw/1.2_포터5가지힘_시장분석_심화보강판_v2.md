## 1. 시장 범위 정의

본 보고서는 **국내 중소/중견 제조업체 및 소상공인을 고객으로 하는 AI 기반 사무자동화(OA), RPA, 에이전트 AI(Agentic AI) 도입 컨설팅 및 솔루션 개발·공급 SI 시장**을 분석 대상으로 한다.

### 시장 규모 (추정)

| 구분 | 2023년 | 2027년 (전망) | CAGR |
|------|--------|--------------|------|
| 국내 RPA 전체 시장 | 약 1,500억 원 | 약 4,000억 원 | ~27% |
| 중소기업 특화 SAM (추정 20~30%) | 약 300~450억 원 | 약 800~1,200억 원 | ~28% |
| 정부 AI/DX 지원 예산 | 약 8,000억 원 | 약 1.5조 원 (전망) | ~17% |

## 2. 포터의 5가지 힘 분석

*(기존 분석 내용 유지 — 요약 버전)*

| Force | 강도 | 핵심 근거 |
|-------|------|-----------|
| **F1. 신규 진입자 위협** | 🔴 **High** | LLM API 기술 장벽 붕괴, 낮은 초기 자본, 프리랜서·에이전시 급증 |
| **F2. 공급자 교섭력** | 🟡 **Medium** | AI 인력 수급 불균형이 핵심, LLM·클라우드는 다수 선택지로 분산 |
| **F3. 구매자 교섭력** | 🟡 **Medium** | 가격 민감도 高, 정보 비대칭성과 도입 후 전환비용이 공급자 우위 견인 |
| **F4. 대체재 위협** | 🟡 **Medium** | 범용 SaaS 기초 잠식, 맞춤형 연동·한국어 지원 영역은 대체 불가 |
| **F5. 기존 경쟁자 경쟁** | 🔴 **High** | 기능 동질화 + 가격 경쟁 심화 + 바우처 단가 압박 |

**종합 시장 매력도**: ⭐⭐⚡ **2.6 / 5.0** (중하 — 진입은 쉽지만 수익성 확보는 어려움)

## 3. 종합 강도 매트릭스 및 시장 매력도

### 핵심 기회 요인 TOP 3
1. **블루오션 틈새**: 대형 SI는 단가 미스매치, 기존 ERP사는 AI 역량 부족 → "AI 아는 중소기업 전문가" 희소
2. **정책 수요 창출**: 연 1조 원 이상 정부 지원금이 수요를 인위적으로 촉진
3. **구인난·인건비 상승**: 자동화 수요의 구조적·자생적 성장 기반 확보

### 핵심 위협 요인 TOP 3
1. **기능 동질화**: 솔루션 표준화로 가격 경쟁 불가피
2. **빅테크 직접 침투**: MS Copilot, Google Workspace AI 번들링으로 기초 수요 잠식
3. **AI 인력 확보 경쟁**: 우수 인력 이탈 시 서비스 품질·납기 직격

## 보강 A. 고객 페르소나 관점 분석

> **목적**: 추상적 "중소기업"을 실제 의사결정자별로 분해하여, 각 페르소나의 Pain Point·구매 기준·접근 방식을 구체화한다.

---

### 페르소나 1: 제조업 대표 "이공장 사장님" (50대, 금속부품 중소 제조업)

| 항목 | 내용 |
|------|------|
| **역할** | 대표이사 (의사결정 100% 본인) |
| **종업원 수** | 30~80명 |
| **주요 Pain Point** | 경리 담당자가 매월 수기로 세금계산서 입력 중. 발주서·납품서 관리에 하루 2시간 낭비. 구인이 안 됨 |
| **AI 인식 수준** | "ChatGPT 들어봤지만 우리 회사에 어떻게 적용하는지 모름" |
| **구매 기준** | 돈이 얼마나 드는가? → 정부 지원받을 수 있는가? → 효과가 눈에 보이는가? |
| **구매 저항** | 직원들이 배울 수 있을까? / 도입 후 AS는 누가 해주나? / 사기 아닌가? |
| **접근 채널** | 지인 소개, 중소기업 협회, 상공회의소 세미나, 지역 산업단지 |
| **이상적 솔루션** | "우리 회사 현장을 직접 와서 봐주고, 국어로 설명해주고, 정부지원 받게 도와줘" |

**→ 전략적 함의**: 기술 설명 금지. ROI 계산기 제시, 바우처 신청 대행 서비스, 업종 유사 레퍼런스 케이스 필수.

---

### 페르소나 2: 중견 제조업 IT 담당자 "박과장" (30대, 자동차 부품 중견사)

| 항목 | 내용 |
|------|------|
| **역할** | IT·전산팀 과장 (도입 검토 실무자, 최종 결재는 임원) |
| **종업원 수** | 300~500명 |
| **주요 Pain Point** | ERP(더존/SAP)와 엑셀 사이의 수작업 복사-붙여넣기가 전사 비효율의 원인. 경영진이 "AI 도입하라"고 지시했지만 어디서부터 시작해야 할지 모름 |
| **AI 인식 수준** | "LLM, RPA 개념은 이해함. 내부 구축은 어렵고 외부 도입이 현실적" |
| **구매 기준** | 기존 ERP/시스템 연동 가능한가? / 보안·데이터 주권 이슈 없는가? / 유지보수 SLA |
| **구매 저항** | 도입 후 내부 반발(자동화로 자기 일 없어지는 것 두려워하는 팀원) / 예산 확보 |
| **접근 채널** | IT 전문 미디어, 산업전시회(스마트팩토리+오토메이션월드), 레퍼런스 사이트 직접 검색 |
| **이상적 솔루션** | "기술 문서 제대로 된 벤더. ERP 연동 케이스 있는 곳. PoC 먼저 해보고 싶음" |

**→ 전략적 함의**: 기술 문서·API 연동 스펙 필수. PoC 선 제공 전략 유효. 보안 인증(ISO, ISMS) 취득이 차별화.

---

### 페르소나 3: 소상공인 "김원장" (40대, 피부/헤어샵 운영)

| 항목 | 내용 |
|------|------|
| **역할** | 대표 겸 실무자 (예약, 고객관리, SNS, 세무 등 혼자 담당) |
| **종업원 수** | 1~5명 |
| **주요 Pain Point** | 예약 노쇼, 단골관리, 인스타 포스팅, 세금계산서 처리에 하루 3시간 낭비 |
| **AI 인식 수준** | "ChatGPT로 SNS 글 써봤음. 그 이상은 어려움" |
| **구매 기준** | 비용이 월 몇 만 원 이내 / 나 혼자 쓸 수 있는가 / 지금 당장 효과 있는가 |
| **구매 저항** | "IT는 어려워서..." / 계약하면 나중에 끊기 어렵지 않나? |
| **접근 채널** | 인스타그램·유튜브 광고, 소상공인 지원센터, 같은 업종 원장들 카카오톡 단체방 |
| **이상적 솔루션** | "앱처럼 쉽게 설치하고 바로 쓰는 것. 월 3~5만원이면 시도해볼게요" |

**→ 전략적 함의**: No-code 기반 SaaS 방식 필수. 구독형 월정액. 업종 커뮤니티 바이럴 전략. 7일 무료 체험.

---

### 페르소나 4: 스마트팩토리 고도화 담당 "최부장" (40대, 식품제조 중소기업)

| 항목 | 내용 |
|------|------|
| **역할** | 공장장 겸 스마트팩토리 TF 팀장 |
| **종업원 수** | 100~200명 |
| **주요 Pain Point** | 스마트공장 1단계(기초 자동화) 완료. 이제 데이터 분석·불량 예측·문서 자동화로 가고 싶은데 어디에 어떻게 해야 할지 로드맵이 없음 |
| **AI 인식 수준** | "설비 데이터는 수집 중. AI 연결하면 뭔가 될 거 같은데 구체적 그림이 안 그려짐" |
| **구매 기준** | 현재 MES·ERP와 데이터 연동 가능한가? / 중기부 스마트공장 고도화 지원사업 연계 가능한가? |
| **구매 저항** | 과거 SI 프로젝트 실패 경험으로 불신 / 직원 교육 부담 |
| **접근 채널** | 스마트공장 지원센터, 업종 협회(식품산업협회 등), 중기부 연계 컨설팅 기관 |
| **이상적 솔루션** | "우리 공장 데이터 보고 AI 적용 로드맵 그려주는 컨설턴트 + 단계별 구현" |

**→ 전략적 함의**: 컨설팅 선행 → 구현 후행 방식. 스마트공장 고도화 사업 공급사 등록 필수. 데이터 파이프라인 구축 역량 필요.

## 보강 B. 신규 진입 가능 영역 vs. 어려운 영역 구분

> **목적**: 자원이 제한된 신규 진입자(스타트업/초기 SI)가 어디에 먼저 집중해야 하는지 현실적으로 판단한다.

---

### ✅ 진입 가능 영역 (문이 열려 있다)

| 영역 | 이유 | 대표 서비스 예시 |
|------|------|----------------|
| **문서 자동화·사무 RPA** | LLM API로 빠른 구현 가능, 고객 Pain이 즉각적이고 명확함 | 세금계산서 자동입력, 발주서 자동생성, 이메일 분류·답장 |
| **소규모 제조업 스마트팩토리 컨설팅** | 대형 SI 진입 無, 정부 사업 바우처 연계 가능, 레퍼런스 확보 용이 | 공정 데이터 수집 → 불량예측 대시보드 구축 |
| **소상공인 업종 특화 SaaS** | 경쟁자 희소, 월정액 수익 모델 가능, 업종 커뮤니티로 바이럴 확산 | 헤어샵 예약·노쇼 방지 AI, 음식점 발주 자동화 |
| **업종 특화 AI 챗봇 구축** | 개발 기간 짧음(2~4주), 정부 바우처 연계 쉬움 | CS 자동응대 챗봇, 사내 FAQ 봇, 영업 문의 1차 대응 봇 |
| **기존 ERP 데이터 AI 분석 레이어** | 더존·영림원 ERP 데이터 API 연동 노하우 확보 시 강한 경쟁우위 | 매출 이상 감지 알림, 재고 자동발주 트리거, 인사 근태 분석 |
| **정부사업 연계 AI 컨설팅** | 바우처 공급사 등록만으로 고객 유입, 영업비용 제로 | AI 바우처, DX 바우처, 스마트공장 고도화 사업 |

---

### ❌ 진입 어려운 영역 (지금 당장은 피해야 할 곳)

| 영역 | 장벽 | 현실적 대안 |
|------|------|------------|
| **대기업·공공 SI 입찰** | 조달청 실적 요건, 기술등급, 최저가 경쟁 지옥 | 중소기업 전용 바우처로 우회 |
| **글로벌 RPA 플랫폼 재판매** | UiPath·AA 라이선스 마진 박함, 가격 경쟁에서 대리점 불리 | 오픈소스(n8n) + LLM 직접 구현으로 대체 |
| **ERP 자체 개발 및 공급** | 구축 기간 6~18개월, 안정성 레퍼런스 필수, 자본 소요 大 | ERP 위에 AI 레이어 추가하는 방식으로 접근 |
| **제조 MES·PLC 직접 연동** | 현장 하드웨어 전문 지식 필요, 안전 규정·인증 복잡 | MES 데이터 수집 이후 단계(소프트웨어 레이어)만 담당 |
| **대규모 데이터센터·인프라 구축** | 초기 자본 수십억 원 이상, 운영 인력 필요 | 클라우드 완전 위탁 구조로 설계 |
| **AI 모델 자체 개발(파운데이션)** | 수십억~수백억 원 투자 필요, 빅테크와 정면 경쟁 불가 | LLM API 활용 애플리케이션 레이어에만 집중 |

### 🟡 조건부 진입 가능 영역 (특정 조건 충족 시)

| 영역 | 진입 조건 |

## 보강 D. 핵심 성공요인(KSF) 및 차별화 전략

> **KSF(Key Success Factor)**: 이 시장에서 살아남고 성장하기 위해 반드시 갖춰야 할 역량

---

### KSF 1: 도메인 깊이 (Domain Depth) — 最重要

**왜 중요한가?**
기능이 동질화된 시장에서 유일한 탈출구는 "AI + 업종 전문성" 조합이다.
경쟁자는 AI 기술을 복사할 수 있지만, **특정 업종 10년의 노하우는 복사되지 않는다**.

**달성 방법**:
- 특정 업종(예: 금속가공, 식품, 의류) 전문가를 공동창업자/핵심 멤버로 확보
- 업종 협회·조합 임원과의 네트워크 구축
- 업종별 업무 프로세스 사전 매핑(Standard Process Map) 자산화

---

### KSF 2: 레퍼런스 선점 (Reference Monopoly)

**왜 중요한가?**
중소기업 구매 의사결정의 핵심은 "우리랑 비슷한 회사가 써봤냐"이다.
업종 내 첫 번째 레퍼런스를 구축한 업체가 이후 연쇄 수주를 독점하는 경향이 강하다.

**달성 방법**:
- 초기 3~5개사 무료 또는 최소 비용 PoC 제안 → 성과 달성 후 케이스 스터디 작성
- 지역 산업단지·업종 협회 세미나에서 성공 사례 발표 → 입소문 영업
- "○○업계 AI 자동화 1호 도입 사례" 타이틀 적극 활용

---

### KSF 3: 정부 사업 접근성 (Gov-Access)

**왜 중요한가?**
이 시장의 수요 창출은 정부가 주도한다. 바우처 공급사가 아니면 사실상 영업 불가 상태에 가깝다.

**달성 방법**:
- NIPA AI 바우처 공급기업 등록 (연 2회 공모)
- 중기부 디지털 전환 바우처 공급사 등록
- 스마트공장 고도화 사업 전문기관 등록
- 지역 테크노파크·창업진흥원과 MOU 체결

---

### KSF 4: 구현 속도와 반복 가능성 (Speed & Repeatability)

**왜 중요한가?**
단가가 낮은 중소기업 시장에서 수익을 내려면 **같은 솔루션을 빠르게 반복 납품**하는 구조가 필수다.

**달성 방법**:
- 업종별 자동화 "템플릿 라이브러리" 구축 → 신규 수주 시 커스터마이징만으로 납품
- LangChain, n8n 기반 에이전트 프레임워크 표준화로 개발 기간 단축
- 납품 기간 목표: 첫 번째 고객 6주 → 다섯 번째 고객 2주

---

### KSF 5: 고객 성공 관리 (Customer Success)

**왜 중요한가?**
중소기업은 도입 후 사용이 제대로 안 되는 경우가 많다. 사용이 안 되면 LTV가 0이 되고 부정적 입소문이 퍼진다.

**달성 방법**:
- 납품 후 30일 무상 현장 지원 표준화
- 월 1회 활용 현황 리포트 발송
- 고객 담당자 교육 매뉴얼 한국어·영상 제공
- "AI 고도화 로드맵" 정기 제안으로 추가 수주 연결

---

| 차별화 축 | 전략 | 기대 효과 |

## 보강 E. 3가지 시나리오 분석

---

### 시나리오 개요

| 구분 | 낙관 시나리오 🟢 | 기준 시나리오 🟡 | 비관 시나리오 🔴 |
|------|----------------|----------------|----------------|
| **가정** | 정부 지원 확대 + 제조업 AI 수요 급증 | 현재 추세 유지 | 경기침체 + 고객 투자 축소 |
| **시장 성장률** | CAGR 35~40% | CAGR 25~30% | CAGR 5~10% or 역성장 |
| **정부 지원 예산** | 연 1.5~2조 원 | 연 약 1조 원 (현수준) | 예산 삭감 30~50% |
| **발생 가능성** | 25% | 50% | 25% |

---

### 🟢 낙관 시나리오: 정부 지원 확대 + 제조업 AI 수요 급증

**배경 가정**:
- 정부가 2026~2027년 "AI 전국민화" 2단계 사업을 본격 추진
- 미·중 무역 갈등으로 국내 제조업 리쇼어링 가속, 생산성 향상 수요 급증
- LLM 성능 향상으로 AI 자동화의 실질 ROI가 극명하게 증명됨

**경쟁 강도 변화**:
- 수요 급증으로 파이가 커져 경쟁 강도는 단기 완화
- 그러나 2~3년 후 자본력 갖춘 중견 SI·빅테크의 시장 진입 본격화로 재심화
- 선점 업체와 후발 업체 간 격차 빠르게 벌어짐

**고객 수요 변화**:
- 도입 검토 기간 단축 (평균 6개월 → 2개월)
- 단순 자동화 넘어 "AI 에이전트 기반 의사결정 지원" 수요 발생
- 고객의 AI 이해도 향상으로 요구사항이 구체화·고도화됨

**살아남는 사업모델**:
- ✅ 업종 특화 AI 솔루션 (속도 > 완성도)
- ✅ 구독형 SaaS (MRR 기반 반복 수익)
- ✅ 스마트팩토리 AI 고도화 (단가 높고 경쟁 적음)
- ❌ 단순 문서 자동화 단품 (범용화로 가치 하락)

**우리가 취해야 할 전략**:
1. **속도 최우선**: 레퍼런스 선점이 미래 독점을 결정. 지금 당장 첫 번째 고객부터
2. **SaaS 전환 준비**: 프로젝트 수익으로 특정 업종 SaaS 개발에 투자
3. **파트너십 확장**: 바우처 사업 재확대 시 공급사 등록 추가, 지역 거점 확보
4. **인력 선확보**: 고성장 시 AI 인력 품귀 극심. 지금부터 내부 육성 프로그램 가동

---

### 🟡 기준 시나리오: 현재 추세 유지

**배경 가정**:
- 정부 지원 사업 현 수준 유지 (연 약 1조 원)
- 제조업 디지털 전환 수요 꾸준히 증가하나 급격한 변화 없음
- AI 기술 발전 지속, LLM 비용 점진 하락

**경쟁 강도 변화**:
- 신규 진입자 계속 유입으로 경쟁 강도 점진 상승
- 가격 경쟁 지속 심화로 범용 RPA/자동화 시장의 평균 단가 하락 추세
- 업종 특화 플레이어만 마진 유지 가능

**고객 수요 변화**:
- AI 도입 인식 향상으로 초기 교육 비용 감소
- 도입 후 효과에 대한 기대치 상승으로 요구 수준 높아짐
- 정부 지원금 없이는 도입 결정을 못 하는 구조 지속

**살아남는 사업모델**:
- ✅ 버티컬 특화 + 반복납품 (수익성 확보)
- ✅ 유지보수 기반 MRR (안정 현금흐름)
- ✅ 컨설팅-구현 번들 패키지 (경쟁 우위)
- ❌ 순수 저가 RPA 단품 납품 (마진 붕괴)

**우리가 취해야 할 전략**:
1. **수익성 관리 최우선**: 모든 프로젝트에 유지보수 계약 필수 포함
2. **레퍼런스 자산화**: 성공 사례를 마케팅 콘텐츠로 체계 전환
3. **비용 효율화**: 템플릿 라이브러리 구축으로 납품 단가 유지 하에 원가 절감
4. **2~3개 업종 집중**: 에너지 분산 금물. 선택과 집중으로 업종 내 압도

---

### 🔴 비관 시나리오: 경기침체 + 고객 투자 축소

**배경 가정**:
- 글로벌 경기침체로 국내 중소기업 투자 심리 급랭
- 정부 재정 악화로 AI 바우처·디지털 전환 사업 예산 30~50% 삭감
- 원자재·에너지 가격 상승으로 제조업 수익성 악화, IT 투자 후순위화

**경쟁 강도 변화**:
- 수요 감소 + 공급자 증가 → 경쟁 강도 극심 악화
- 가격 덤핑 만연, 바우처 의존 업체 다수 폐업
- 소수 강자(도메인 깊이·장기 계약 보유)만 생존, 시장 재편

**고객 수요 변화**:
- "AI 도입? 나중에" 심리 만연
- 신규 도입보다 기존 도입 시스템의 비용 절감 요구로 전환
- 정부 지원 없이는 유의미한 수요 사실상 消滅

**살아남는 사업모델**:
- ✅ **구독형 유지보수** (해지 비용이 높아 계약 유지됨)
- ✅ **ROI 확실한 비용절감형 자동화** ("이거 하면 월 300만 원 아낀다" 증명 가능한 것)
- ✅ **정부 의존형 모델** (남은 예산 집중 활용)
- ❌ 고단가 신규 프로젝트 영업 (예산 동결로 의사결정 불가)
- ❌ 소상공인 SaaS (구독 취소 급증)

**우리가 취해야 할 전략**:
1. **현금 확보 최우선**: 신규 수주보다 기존 고객 계약 유지에 집중
2. **비용 절감 메시지로 전환**: "AI = 성장"이 아닌 "AI = 비용 절감·생존"으로 영업 메시지 전환
3. **공공/정부 사업 比重 확대**: 남은 예산이라도 확보하는 전략
4. **규모 축소·핵심 집중**: 비핵심 서비스 中단, 2개 이내 버티컬에 生존 집중
5. **위기를 기회로**: 경쟁사 폐업 시 고객 흡수 기회. 인수합병 검토

## MVP 설계를 위한 보고서 보강 계획

> **목적**: 현재 시장 분석 보고서를 MVP(Minimum Viable Product) 설계의 실질적 기초로 전환하기 위해 추가로 검증·보완해야 할 사항을 로드맵으로 정리한다.

---

### 현재 보고서의 MVP 관점 위치

```
[거시 시장 분석] ← 현재 위치 (完)
       ↓
[고객 페르소나 심화] ← 이번 보강으로 초안 완성
       ↓
[Problem-Solution Fit 검증] ← 다음 단계 필요
       ↓
[Product-Market Fit 가설 수립] ← MVP 설계 직전
       ↓
[MVP 설계 및 PoC 실행] ← 최종 목표
```

---

### Phase 1: 고객 검증 (Customer Discovery) — 즉시 착수

| 활동 | 목표 | 방법 | 기간 |
|------|------|------|------|
| **페르소나 인터뷰** | 실제 Pain Point 수치 검증 | 타겟 10~20개사 대표/담당자 30분 인터뷰 | 4주 |
| **구매 의사 탐침** | "실제로 돈 낼 것인가" 확인 | 가상 견적 제시 후 반응 측정 | 인터뷰 중 |
| **현재 솔루션 현황 파악** | 대체재 위협 실태 | 현재 사용 중인 툴·서비스 목록 수집 | 인터뷰 중 |
| **예산 현실 파악** | 가격 전략 수립 | 연간 IT 예산 실제값 수집 | 인터뷰 중 |

**인터뷰 핵심 질문 5가지**:
1. "현재 가장 많은 시간을 낭비하는 반복 업무는 무엇입니까?"
2. "자동화가 된다면 한 달에 얼마까지 쓸 수 있습니까?"
3. "지금 AI/자동화 도입을 망설이는 가장 큰 이유가 무엇입니까?"
4. "과거 IT 솔루션 도입 경험 중 가장 실망한 사례와 이유는?"
5. "이상적인 AI 자동화 파트너의 조건은 무엇입니까?"

### Phase 2: 핵심 가설 도출 및 검증 계획 수립

인터뷰 결과를 바탕으로 아래 가설들을 검증한다.

| 가설 번호 | 가설 내용 | 검증 방법 | 검증 기준 |
|----------|----------|----------|----------|
| H1 | 중소 제조업의 가장 큰 Pain은 문서/경리 자동화다 | 인터뷰 | 응답자 70% 이상 동의 |
| H2 | 정부 바우처가 없으면 도입 결정이 안 된다 | 인터뷰 | 응답자 60% 이상 확인 |
| H3 | 월 50만 원 이하 구독이면 소상공인도 결제한다 | 랜딩페이지 A/B 테스트 | CTA 클릭률 5% 이상 |
| H4 | ERP 연동 자동화가 단순 자동화보다 단가 3배 높다 | 견적 반응 측정 | 비교 견적 제시 실험 |
| H5 | 레퍼런스 1건이 동종 업계 2~3건 수주로 연결된다 | 기존 고객 referral 추적 | 파일럿 후 검증 |

## 최종 시사점

> 이 시장은 **"진입은 쉽지만 수익화는 어렵고, 차별화만이 살길인 시장"** 이다.

**핵심 결론 4가지**:

1. **버티컬을 먼저 고르라**: 업종을 먼저 정하고, 그 업종의 전문가가 된 다음 AI를 붙여라. 반대 순서는 실패한다.

2. **첫 번째 고객이 사업의 방향을 결정한다**: 초기 레퍼런스 고객 선택이 이후 수주 파이프라인 전체를 결정하므로, 전략적으로 선별하라.

3. **정부 사업은 채널이지 목적이 아니다**: 바우처로 진입하되, 바우처 없이도 고객이 돈을 내는 구조를 반드시 설계해야 지속 가능하다.

4. **어떤 시나리오가 펼쳐져도 "버티컬 특화 + 유지보수 MRR"은 유효하다**: 이것이 이 시장의 불패 전략이다.
