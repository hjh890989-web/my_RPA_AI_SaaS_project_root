# 페르소나별 Pain 중요도 평가 (Impact Assessment)
## 각 Pain이 고객의 목표(Goal) 달성에 미치는 치명도 분석

> **작성 목적**: 12인 페르소나가 겪고 있는 핵심 Pain이 그들의 궁극적 Goal에 도달하는 데 있어 **얼마나 치명적인 장애물(중요도 1~5점)**인지 평가한다. 점수가 높을수록 우리 솔루션이 반드시 해결해야 할 핵심 과제(Must-have)임을 의미한다.
> **평가 기준**:
> * **5점 (치명적/Fatal)**: 이 문제가 해결되지 않으면 Goal 달성이 원천 불가능하며, 비즈니스 존재(또는 개인 커리어)가 위협받는 수준. (필수 해결)
> * **4점 (고위험/High)**: Goal 달성이 심각하게 지연되거나 반쪽짜리 성공에 그침. 강력한 병목 현상.
> * **3점 (중간/Medium)**: 불편함과 비효율이 크지만, 현행 유지(Workaround)로 당장 망하지는 않는 수준.
> * **2점 (저위험/Low)**: 성가시긴 하지만, 다른 수단으로 쉽게 대체 가능함.
> * **1점 (무의미/None)**: Goal 달성과 큰 연관성이 없음.
> **작성일**: 2026년 4월

## 🎯 평가 결과 핵심 브리핑 (Where to Focus)

### 🚨 최우선 해결 과제 (중요도 5점 / Fatal)
이 과제들을 해결하는 기능과 메시지는 제품(MVP)의 **메인 밸류 프로포지션 (Core Value Proposition)**이 되어야 합니다.
1. **인적 스케줄링 의존 (Core-02)**: 납기 지연 방어
2. **원천 데이터 입력 거부 (Core-04)**: 자동 로깅/리포트 필수
3. **투자 실패로 인한 커리어 붕괴 (Core-05)**: 확실한 PoC 및 ROI 대시보드
4. **품질 검증 불신 (Ext-02)**: 원인 추적 가능한 XAI
5. **데이터 외부 유출 정책 (Non-01)**: 폐쇄망/온프레미스 배포
6. **글로벌 공급망 실사 (Adj-02)**: 투명한 제조 이력 트래킹

### ⚠️ 주요 과제 (중요도 4점 / High)
1. **재무적 ROI 확보 (Adj-01)**: 정부 바우처 연계로 해결 (가격 전략)
2. **단절된 데이터 시스템 (Core-03)**: 플러그인 방식의 커넥터로 해결 (사용성 전략)
3. **창업주의 자존심 (Ext-01)**: 도입 메시지의 프레이밍(유산, 어드바이저)으로 해결 (마케팅 전략)

> **전략적 시사점**: 고객들이 가장 중요하게(5점) 느끼는 고통은 **'돈(직접 비용)'보다 '생존(납기, 품질 클레임, 커리어 파괴, 규제 위반)'**에 더 치우쳐 있습니다. 즉, 솔루션을 팔 때 "얼마 아껴줍니다"보다 **"사고 나는 것을 완벽히 막아줍니다" (방패의 역할)**가 더 강력한 세일즈 소구점이 됩니다.
