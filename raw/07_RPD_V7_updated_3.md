## 1. 개요·목표

### 1-1. 문제 정의 (Pain 지표 포함)

국내 스마트공장 도입 기업의 **75.5%(≈24,038개사)**가 기초 단계에 정체되어 있다 (이하 '2차 자동화 공백').  

#### 「2차 자동화 공백(Second Automation Gap)」의 구조적 인과관계

정부 주도 스마트공장 보급 정책으로 **기초 인프라(MES·ERP)는 상당수 도입**되었으나, 현장 작업자가 키오스크·태블릿 입력을 전면 거부하면서 **대시보드가 빈 껍데기**가 되었다. 이로 인해 데이터가 축적되지 않아 AI 고도화 단계로 올라갈 수 없는 구조적 공백이 형성되었으며, 특정 인물 의존(SPOF), 수작업 감사 대응, ERP-MES 분절이라는 악순환이 반복된다.

> **인과 체인**: 정부 보급 → MES/ERP 기초 설치 → 현장 입력 거부(결측률 40%+) → 데이터 공백 → AI 고도화 불가(0.1%) → SPOF·감사 실패·사일로 반복

데이터 인프라(MES·ERP)는 설치되었으나 **현장 활용률 10% 미만**이며, 이로 인해 7가지 구조적 실패가 반복된다.

PoC 담당자(OPERATOR)에게 수동 판단 전환 알림. **XAI 복구 후 30분 이내 자동 복귀**, 수동 판단 기간 중 전 건 감사 로그 기록 | 차품질: "확률적 답변에 극도로 거부감, 100% 확실하지 않으면 안 씀" |

| **맥락** | 품질이사 "AI 오판 책임" 불안 + CISO "AI 단독 제어" 거부가 의사결정 지연의 핵심 원인 |
| **근거** | §3-C HITL 4대 원칙, JTBD ▶10 Anxiety 패턴, ▶7 차품질 프로파일 |
| **결과** | `APPROVAL` 테이블 기반 승인 워크플로, XAI 의무 표시, API 게이트웨이 레벨 차단 |
| **트레이드오프** | 자동화 효율 감소 (모든 AI 결과에 인간 승인 필요) → 품질이사·CISO 신뢰 확보로 계약 성사율 극대화 |

### ADR-4: USB 오프라인 배포

| 항목 | 내용 |
|:---|:---|
| **결정** | 에어갭 환경에서 USB/내부망을 통한 모델 업데이트. 해시 검증 필수 |
| **맥락** | 온프레미스 환경에서 인터넷 연결 없이 모델 업데이트 안전성 확보 필요 |
| **근거** | §6-3 오프라인 데이터 흐름, US-05 AC-2 (오프라인 업그레이드 성공률 100%) |
| **결과** | 본사 빌드 → USB 패키징 → 물리 전달 → 해시 검증 → 설치/거부 워크플로 |
| **트레이드오프** | 업데이트 주기 제한(물리 방문 필요) → 경량 모델 최적화로 업데이트 빈도 최소화 |

### ADR-5: Zero-Touch UX (패시브 수집)
