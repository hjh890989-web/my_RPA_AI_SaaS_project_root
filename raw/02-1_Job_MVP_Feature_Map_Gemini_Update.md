# Job(Value) - MVP Feature Map
## 고객의 핵심 완수 과제(Job)와 제품 기능(Feature) 매핑 보드

> **문서 목적**: 고객이 해결하고자 하는 근본적인 '과제(Job-to-be-Done)'와 이를 달성하기 위한 당사의 '제안 가치(Value)'를 매핑하고, 이를 실제 구현할 **MVP 단위 기능(Feature)들과 그 개발 우선순위**를 직관적으로 나열합니다.

---

## 2. 우선순위에 따른 핵심 기능(Feature) 상세 목록

전체 E1~E7 중 MVP에 반드시 포함되어야 하는 최상위 우선순위 기능(P1, P2) 중심의 세부 기능 트리입니다.

### 🥇 Priority 1 (P1): 제품 본연의 생존과 핵심 가치 창출 (Must Have)

**[Feature 1] 무입력 패시브 로깅 엔진 (Zero-Touch Logging)**
*   **F1.1 노이즈 캔슬레이션 STT 커맨더**: 80dB 이상의 현장 기계음 속에서도 작업자의 지시어("금형 교체", "생산 완료")만 트리거하여 텍스트로 치환하는 모듈. (Whisper API 튜닝)
*   **F1.2 Vision 상태 캡처기**: 모바일 카메라로 공정 완성품/바코드/계기판을 찍으면 LLM이 상태 값을 파싱하여 시스템에 밀어 넣는 기능.
*   **F1.3 로그 롤백/수정 관리자 웹**: AI가 오인식한 데이터를 관리자(반장)가 퇴근 전 한 화면에서 확인하고 일괄 수정(Approve/Reject)할 수 있는 웹 뷰어.

**[Feature 2] 자동화된 추적성(Traceability) 컴플라이언스 봇**
*   **F2.1 파편화 로트(Lot) 머지 로직**: 수집된 로깅 데이터와 기존 ERP 재고 데이터를 로트 번호 기준으로 시간순 병합하는 백엔드 엔진.
*   **F2.2 템플릿 매핑 PDF 제너레이터**: 삼성전자, 현대차, EU CBAM 등 주요 규제 포맷에 맞게 테이블 뷰를 재배치하고 워터마크가 찍힌 무결성 PDF로 다운로드 및 이메일 전송하는 기능.

**[Feature 3] 비파괴형 레거시(ERP) 브릿지**
*   **F3.1 Read-Only DB 커넥터**: 특정 테이블만 읽어올 수 있도록 하여 고객사 IT 담당자의 정보유출/DB손상 핑계를 원천 차단하는 플러그인.
*   **F3.2 Low-Tech 엑셀 Batch 업/다운로더**: API 개방을 절대 불허하는 극보안 타겟사를 위해, 기존 ERP 엑셀 덤프를 드래그 앤 드롭하면 우리 시스템 포맷에 자동 파싱되는 대체 기능.

### 🥈 Priority 2 (P2): 영업 마찰력 제거 및 결재 가속기 (Should Have)

**[Feature 4] CFO 전용 진단 및 도입 시뮬레이터 (B2B 세일즈 내장 툴)**
*   **F4.1 반응형 바우처/ROI 웹 계산기**: 기업의 "직원수, 기존 ERP 명"만 입력하면 현 시점 국가지원금 매칭 확률, 예상 자부담금 축소액, 그리고 야근(시간) 삭감 회수액이 즉시 출력되는 마케팅 및 영업용 무기.

### 🥉 Priority 3 (P3): 고도화 및 장기 비전 (Out of MVP Scope - Phase 2)

**[Feature 5] 완전 자동화 공정 스케줄러**
*   **F5.1 스케줄링 배분 알고리즘**: 특정 1인 담당자의 직관을 시스템으로 교체.
*   *(배제 사유)*: 고객의 목숨을 쥐고 있는 가장 강력한 기회(AOS 4.0 최고점 수요)이나, 현장 변수가 너무 많아 초기 엔진 개발 리소스가 기하급수적으로 들어감(난이도 5). 빠른 MVP 출시(100일) 확보를 위해 부득이하게 Phase 2로 연기.

**[Feature 6] 설명가능 AI (XAI) 품질 감시기**
*   **F6.1 딥러닝 불량 판단 근거 시각화 보드**: AI 추론 원인을 자연어로 제공하여 품질이사의 AI 블랙박스 공포 제어.
*   *(배제 사유)*: 진입 후 패시브 로깅 및 감사 봇을 통해 기본 데이터가 쌓인 이후에야 진정한 위력을 발휘하는 후행 기능이므로 단기 포함 배제.

**[Feature 7] 온프레미스 (폐쇄망) 보안 패키지**
*   **F7.1 CISO 전용 로컬 설치팩 및 하이브리드 구성**: 클라우드를 극단적으로 거부하는 기업용 오프라인 모듈.
*   *(배제 사유)*: 극히 보수적인 일부 기업(CISO 관문 방어 심의사) 대상의 선별적 수요이므로, 도입 난이도가 낮은 퍼블릭 클라우드 가능 타겟(SEG-C)부터 선 공략하기 위함.

## 3. 기능 우선순위(Priority) 산정 원칙 근거

| 결정 요소 | P1 (Must Have) 산정 이유 | P3 (Out of Scope) 배제 이유 |
| :--- | :--- | :--- |
| **비즈니스 임팩트 (JTBD)** | '현장 무입력 체제 유지'와 '원청사 규제/감사 방어'는 당장 발등의 불인 치명적 요건(AOS 4.0 전후)이므로 타협 불가. | '스케줄러 1인 의존(SPOF)' 해소는 생존 직결되는 문제(AOS 4.0 최상위)이나, 불특정 다수의 현장 변수로 인해 초기 MVP의 범위를 넘어섬. 당면 목표인 'Quick-Win 시연' 측면에서 P1보다 후순위. |
| **고객의 지불의사(WTP)** | '자부담 최소화'가 WTP의 가장 강력한 초기 트리거이므로, ROI 계산기(P2)가 결재를 앞당기는 핵심 지렛대 역할을 함. | XAI 모듈 및 폐쇄망(온프레미스) 구축은 초기 도입 직후의 '결과적 불만'을 해소하는 영역이므로 Upsell 모델 구조로 활용함. |
| **타임투마켓 (개발 복잡도)** | PDF 템플릿 생성이나 기존 LLM API 파싱 로직 등은 시장 입증이 마쳐진 구조로 매우 단시간 12주 내 출시(Go-to-market)가 가능함. | 자체 공정 스케줄러(난이도 5)는 기업별 변수로 인해 엔진 구축과 PoC 안정화에만 기본 6개월이 소요됨. 자본을 태우는 핵심 리스크로 정의됨. |
