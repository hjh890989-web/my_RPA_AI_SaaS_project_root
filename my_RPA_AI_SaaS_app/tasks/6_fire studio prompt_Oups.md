# FactoryAI — Firebase Studio UI 프로토타이핑 프롬프트

## 프로젝트 개요

**FactoryAI**는 중소 제조공장(금속가공·식품제조)을 위한 AI 기반 생산관리 SaaS 플랫폼입니다.
작업자 추가 입력 없이 음성(STT)·이미지(Vision)로 현장 데이터를 수집하고, AI가 자동 구조화한 뒤
반드시 인간 승인(HITL)을 거쳐야만 외부 발행되는 "AI 단독 실행 0건" 원칙의 제조 품질관리 시스템입니다.

### 사용자 역할 (5역할 RBAC)
| 역할 | 페르소나 | 핵심 업무 |
|:---|:---|:---|
| ADMIN | 한성우 COO | 전체 관리, ERP 동기화, 대시보드 발행 |
| OPERATOR | 박작업 작업자 | 음성/카메라 로깅, 엑셀 업로드 |
| AUDITOR | 클레어 리 품질이사 | 감사 리포트 승인, XAI 검토, 품질 판단 |
| VIEWER | 이뷰어 열람자 | 읽기 전용 조회 |
| CISO | 최보안 보안책임자 | 보안 콘솔, 감사 로그, 네트워크 모니터링 |

### 대상 업종 (2개 버티컬)
- **금속가공** (대한금속 주식회사, 85명, 안산)
- **식품제조** (한국식품 주식회사, 120명, 천안)

### 3. 패시브 로깅 — 데이터 수집 (`/log-entries`)
- **접근**: ADMIN, OPERATOR, AUDITOR, VIEWER(읽기)
- **상단 탭**: `음성 녹음` | `카메라 촬영` | `엑셀 업로드`

#### 3-1. 음성 녹음 탭
- 큰 🎙️ 마이크 아이콘 버튼 (누르면 펄스 애니메이션)
- 녹음 상태 시각화: 오디오 데시벨 웨이브 애니메이션
- [전송] 버튼 → AI 처리 중 프로그레스:
  - 0~5초: "AI가 데이터를 분석 중입니다..."
  - 5~15초: "심층 모델이 현장 노이즈를 제거하고 있습니다..."
  - 15초+: "요청량이 많아 순차 처리 중입니다. 최대 1분 소요될 수 있습니다."
- 완료 시: ✅ "로그가 PENDING 상태로 저장되었습니다" 토스트

#### 3-2. 카메라 촬영 탭
- 📷 촬영 버튼 (모바일 카메라 호출)
- 촬영 이미지 썸네일 미리보기
- [분석 요청] 버튼 → AI 처리 프로그레스
- 실패 시: ⚠️ "화질 불량 — 재촬영해 주세요" 경고 토스트

#### 3-3. 엑셀 업로드 탭
- 드래그&드롭 영역 ("xlsx 또는 csv 파일을 여기에 놓으세요")
- 파일 선택 후 파싱 결과 DataTable 미리보기:
  - 정상 행: 기본 배경
  - 실패 행: 🔴 빨간 배경 + 에러 사유 컬럼
  - 하단: "총 45행 / 정상 43행 / 실패 2행"
- [적재 확인] 버튼
- 50MB 초과 시: "파일이 너무 큽니다 (52.3MB). 최대 50MB" 에러

### 4. HITL 승인 뷰어 — 롤백/수정 (`/log-entries/review`)
- **접근**: ADMIN, AUDITOR
- **좌측: PENDING 목록** (DataTable)
  - 컬럼: 시간 | 유형(STT/VISION/EXCEL) | 라인명 | 상태 뱃지 | 검토자
  - PENDING 건 상단 정렬, 🔴 뱃지
  - 30분 초과 건: ⏰ 아이콘 + 빨간 테두리
- **우측: 상세 패널** (행 클릭 시)
  - 상단: 원본 데이터 (오디오 플레이어 또는 이미지 뷰어)
  - 중단: AI 추출 결과 JSON → 편집 가능 폼 (온도, 압력, 상태 등)
  - 하단 버튼:
    - `✅ 승인 (Approve)` — 녹색
    - `❌ 거절 (Reject)` — 빨간색 (사유 입력 필수)
  - 감사 로그 타임라인 (생성 → AI 파싱 → 검토자 판단)

### 5. 감사 리포트 (`/audit-reports`)
- **접근**: ADMIN, AUDITOR, VIEWER(읽기)

- 성공 시: PDF 미리보기 (iframe)
- 결측치 감지 시: 🔴 모달 "다음 항목이 기록되지 않았습니다: [온도, PH농도]" + 수동 입력 폼

#### 5-2. 리포트 목록/이력

- PENDING 건: "품질이사 승인 대기" 뱃지
- 행 클릭 → PDF 미리보기 + XAI 판단 의견 블록

### 6. XAI 이상탐지 대시보드 (`/dashboard/xai`)
- **접근**: ADMIN, AUDITOR, OPERATOR

#### 6-1. 좌측: 이상 징후 목록 (DataTable)
- 컬럼: 시간 | 라인명 | 이상 유형 | 심각도(INFO/WARNING/CRITICAL) | 상태(PENDING/JUDGED)
- PENDING 건: 🔴 빨간 뱃지, 최상단 정렬
- 수동 판단 건: 🟡 노란 뱃지 + "수동 판단" 태그

#### 6-2. 우측: AI 설명 상세 패널
- **AI 설명 모드** (정상):
  - 한국어 설명 블록 (큰 폰트): "CNC-3번 라인 온도가 설정값(180°C) 대비 2.3°C 높습니다. 과거 유사 패턴에서 불량률 12% 상승이 관측되었습니다."
  - 데이터 하이라이팅 테이블: 이상 값에 노란 배경 + 빨간 테두리
  - 30일 추세 미니 차트 (현재 값 마커)
  - 권장 조치 리스트 (체크박스)
- **수동 판단 모드** (XAI 실패):
  - ⚠️ 경고 배너: "AI 설명 생성 불가 — 원본 데이터 기반 수동 판단을 요청합니다"
  - 원본 데이터 테이블 (항목 | 감지 값 | 설정 기준 | 편차)
  - 판정 드롭다운: `정상` | `이상 — 조치 필요` | `판단 보류`
  - 사유 텍스트 (필수, 최소 10자)

#### 6-3. 하단 액션 바
- `✅ 정상 판단 — 승인` | `❌ 이상 확인 — 조치 필요` | `⏸️ 보류`

#### 6-4. 판단 이력 타임라인
- 세로 타임라인: AI 감지 → XAI 설명 생성 → 인간 판단 → 최종 결과
