- **목적**: FactoryAI의 최상위 안전 원칙인 **"AI 단독 실행 0건"**을 검증한다. 인간의 명시적 승인/거절 없이는 어떤 고위험 액션(STOP, CHANGE, 발행)도 실행되지 않으며, 모든 시도가 100% 차단되고 보안 이벤트로 기록됨을 보증한다. 이는 HITL(Human-In-The-Loop) 4대 원칙의 핵심 검증이며, 프로젝트 전체의 안전성을 좌우하는 **최우선순위 테스트**이다.

### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: AI 이상 감지 결과 → 미승인 상태에서 자동 실행 시도 → 100% 차단**
- **Given**: XAI 모듈이 이상 징후를 감지하고 "설비 정지(STOP)" 권장 조치를 생성했으나, 품질이사의 승인이 없음 (`APPROVAL.status=PENDING`)
- **When**: 시스템이 STOP 액션을 자동 실행하려 시도함
- **Then**: 즉시 차단(≤1초)되고, 403 Forbidden이 반환되며, `AUDIT_LOG`에 "미승인 자동 실행 차단" 이벤트가 기록되고, CISO에게 ≤10초 내 알림이 발송된다. **AI 단독 실행 건수 = 0건**.

**Scenario 2: PENDING 리포트 외부 공유 시도 → 차단**
- **Given**: `status=PENDING`인 감사 리포트가 존재함
- **When**: 외부 공유 또는 다운로드를 시도함
- **Then**: 발행이 즉시 차단되고, ADMIN에게 ≤10초 내 알림이 발송된다.

**Scenario 3: API 직접 호출 우회 시도 → 차단 + CISO 긴급 알림**
- **Given**: `action_type=CHANGE`, `approval_id` 없이 API를 직접 호출함
- **When**: API 게이트웨이에서 승인 검증을 수행함
- **Then**: 즉시 차단(≤1초)되고, `AUDIT_LOG`에 "우회 시도 감지"가 기록되며, CISO에게 긴급(CRITICAL) 등급 알림이 발송된다.

**Scenario 4: 승인 시스템 장애 시 → Fail-Close (기본 차단)**
- **Given**: APPROVAL 데이터베이스 또는 승인 서비스가 일시 장애 상태임
- **When**: AI 결과물에 대한 발행/실행 요청이 발생함
- **Then**: Fail-Close 원칙에 따라 모든 요청이 차단(기본 거부)되며, "승인 시스템 점검 중" 안내가 반환된다.

**Scenario 5: 정상 승인 후 → 실행 허용 (대조군)**
- **Given**: 품질이사가 `APPROVED`로 승인을 완료한 상태
- **When**: 해당 액션의 실행을 요청함
- **Then**: 정상적으로 실행되고, `AUDIT_LOG`에 "승인 후 실행" 이벤트가 기록된다.

- **When**: 에스컬레이션 스케줄러가 30분 시점에 체크함
- **Then**: `APPROVAL.status=APPROVED`이므로 에스컬레이션이 발동하지 않는다.

**Scenario 4: 이중 에스컬레이션 방지**
- **Given**: 이미 COO에게 에스컬레이션 알림이 발송된 건 (`escalation_status=ESCALATED_TO_COO`)
- **When**: 다음 스케줄러 주기에 동일 건이 다시 감지됨
- **Then**: 중복 알림이 발송되지 않는다 (COO 알림 = 해당 건당 **최대 1회**).

**Scenario 5: 에스컬레이션 후 품질이사 뒤늦은 응답 수용**
- **Given**: COO 에스컬레이션 후 품질이사가 뒤늦게 `REJECTED`로 응답함
- **When**: 응답이 접수됨
- **Then**: 품질이사의 판단이 정상 반영되고, `escalation_status`에 "에스컬레이션 후 원담당자 응답"이 기록된다.
