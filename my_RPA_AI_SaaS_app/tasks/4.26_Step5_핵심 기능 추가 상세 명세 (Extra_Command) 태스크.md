# Step 5. 핵심 기능 추가 상세 명세 (Extra_Command) 태스크

이 문서는 FactoryAI MVP의 5개 핵심 CMD(Command) 태스크에 대한 통합 상세 명세서입니다. 
각 태스크는 SRS 기반의 기술적 제약, 실행 계획(Task Breakdown), 합격 기준(AC), 그리고 완료 정의(DoD)를 엄격하게 정의합니다.

## 구성 안내
1.  **E1-CMD-001**: 버튼 녹음 → Gemini STT 음성 변환 및 저장
2.  **E1-CMD-002**: 카메라 촬영 → Gemini Vision 이미지 파싱 및 저장
3.  **E2-CMD-001**: Lot 시간순 병합 로직 (타임스탬프 검증)
4.  **E3-CMD-001**: Mock ERP Read-Only 동기화 (3중 Write 차단)
5.  **HITL-CMD-001**: PENDING 발행 자동 차단 (안전 게이트웨이)

---

# [E1-CMD-001] STT 음성 변환 및 저장

### :dart: Summary
- **목적**: 작업자의 음성을 Gemini STT로 텍스트화하여 `LOG_ENTRY`에 `PENDING` 상태로 저장. 결측률을 5% 이하로 낮추는 패시브 로깅의 핵심.
- **핵심 로직**: PCM 16kHz 오디오 전처리 → Vercel AI SDK 호출 → Rate Limiter(15 RPM) 연동 → Prisma 저장.

### :test_tube: Acceptance Criteria (핵심)
- **정확도/지연**: 인식 정확도 ≥85%, 전체 지연 ≤5,000ms (p95).
- **에러 핸들링**: 15 RPM 초과 시 큐잉 처리(최대 60초 대기), 타임아웃 시 1회 재시도.
- **보안**: OPERATOR 이상 권한 필수, 전수 감사 로그 기록.

---

# [E1-CMD-002] Vision 이미지 파싱 및 저장

### :dart: Summary
- **목적**: 촬영된 이미지에서 상태 값(온도, 압력 등)을 Gemini Vision으로 추출하여 `LOG_ENTRY`에 저장.
- **핵심 로직**: 이미지 전처리(≤10MB) → Gemini Multimodal 호출 → Confidence Score(0.5) 기반 파싱 실패 감지.

### :test_tube: Acceptance Criteria (핵심)
- **성능/무결성**: 전 과정 지연 ≤8,000ms (p95). 파싱 실패(confidence < 0.5) 시 저장 차단 및 재촬영 알림 이벤 트 발행.
- **보안**: 비인가 접근 차단 및 감사 로그 연동. Supabase Storage 연동.

---

# [E2-CMD-001] Lot 시간순 병합 로직

### :dart: Summary
- **목적**: 감사 리포트 생성을 위해 파편화된 로그를 LOT 단위로 병합. 타임스탬프 중복/역전을 검증하여 오병합 0건 보장.
- **핵심 로직**: `captured_at` 기준 정렬 → 중분/역전/갭(>30분) 검증 엔진 실행 → 충돌 시 자동 병합 차단 및 수동 해결 워크플로 제공.

### :test_tube: Acceptance Criteria (핵심)
- **정합성**: Lot 정확도 ≥99%, 오병합 0건.
- **성능**: 100개 LOT (총 5,000건) 병합 처리 ≤10초 완료.
- **필터링**: APPROVED 상태의 데이터만 병합 대상 포함.

---

# [E3-CMD-001] Mock ERP Read-Only 동기화

### :dart: Summary
- **목적**: 외부 ERP를 절대 손상시키지 않는 비파괴형 브릿지 구현 (3중 Write 차단).
- **핵심 로직**: `IERPConnector` 인터페이스(Write 배제) → Prisma Middleware(Write 감지) → Supabase RLS 적용.

### :test_tube: Acceptance Criteria (핵심)
- **방어**: 시스템 레벨 Write 시도 발견 시 100% 차단 및 CISO 알림(≤10초).
- **정규성**: 화이트리스트(`approved_tables`) 기반 접근. 스키마 변경 감지 시 동기화 즉시 중단 및 CIO 알림.

---

# [HITL-CMD-001] PENDING 발행 자동 차단 (안전 게이트웨이)

### :dart: Summary
- **목적**: 인간 승인 없이 AI 결과물이 외부로 발행되는 것을 시스템적으로 차단하여 "AI 단독 실행 0건" 보장.
- **핵심 로직**: 중앙 발행 게이트 미들웨어 (`Publication Gate`) → Fail-Close 원칙 적용.

### :test_tube: Acceptance Criteria (핵심)
- **차단 SLA**: 발행 시도 감지 시 즉시 차단(≤1초) 및 관리자 알림(≤10초).
- **무결성**: APPROVAL 상태가 PENDING/REJECTED/MISSING인 모든 케이스 차단. 우회 시도 감지 시 CISO 즉시 알림.
