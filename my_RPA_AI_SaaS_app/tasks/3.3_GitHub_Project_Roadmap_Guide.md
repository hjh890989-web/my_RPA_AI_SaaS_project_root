# GitHub Project — Roadmap 뷰 설정 가이드

**대상 프로젝트**: [my_RPA_AI_SaaS_project #7](https://github.com/users/hjh890989-web/projects/7)
**전제 조건**: `scripts/setup-github-project.ps1` 실행 완료 (212 이슈 + Start/End Date 설정됨)

> [!IMPORTANT]
> GitHub Project V2의 Roadmap 뷰는 **UI에서 수동으로 한 번 설정**해야 합니다 (gh CLI로 자동 설정 불가). 약 2분 소요.

---

## 1. Roadmap 뷰 활성화 (3단계)

### Step 1 — Roadmap 뷰 진입
1. 브라우저: https://github.com/users/hjh890989-web/projects/7
2. 상단 탭 메뉴에서 **"Roadmap"** 클릭
3. 표시되는 알림 `"프로젝트를 시작하려면 최소한 하나의 날짜 또는 반복 필드가 필요합니다"` → **알았어요!** 클릭

### Step 2 — Date 필드 선택
1. 우측 상단 ⚙️ **보기** 버튼 클릭
2. **"필드"** 섹션에서:
   - **"Start date"** → `Start Date` (스크립트가 생성한 필드)
   - **"Target date"** → `End Date`
3. 자동 저장됨 → Roadmap에 모든 이슈가 가로 막대로 표시됩니다.

### Step 3 — 줌 레벨 조정
1. 우측 상단 `+` `-` 버튼으로 줌 조정
2. 권장 줌: **"월"** (5주 일정 전체 한눈에)
3. 또는 **"주"** (주차별 상세)

## 2. 그룹화 옵션 (한눈에 보기 강화)

Roadmap 좌측 패널에서 **Group by**를 변경하여 다양한 뷰:

| Group by | 효과 |
|---|---|
| **(None)** | 모든 이슈 평면 나열 (212건) |
| **Status** | Todo / In Progress / Done 별 분리 (작업 진행도) |
| **Labels — phase:*** | Phase 0/1/2/3/4 별 트랙 (권장) |
| **Labels — epic:*** | Epic E1~E7+HITL+SVC 별 트랙 |
| **Labels — critical-path** | Critical 9건 별도 |

### 권장 첫 설정
- **Group by: `Labels`** → Phase 라벨을 grouping 기준으로 → 5주 진행 흐름 시각화

## 4. 추가 권장 뷰 만들기

### 4-1. "이번 주 To-Do" 뷰 (Board 형식)
1. 좌측 사이드바 `+ New view` 클릭
2. Layout: **Board** 선택
3. Group by: `Status`
4. Filter: `label:phase:1-foundation` (W1) — 매주 phase 변경
5. 이름: "🏃 This Week"

### 4-2. "Critical Path" 뷰 (Table 형식)
1. `+ New view` → **Table**
2. Filter: `label:critical-path`
3. Sort: `Start Date` ascending
4. 이름: "🟥 Critical Path"

### 4-3. "Epic별 진행률" 뷰 (Board)
1. `+ New view` → **Board**
2. Group by: `Labels` → epic:* 선택
3. 이름: "🎯 Epic Progress"

## 5. Status 필드 활용 (선택)

GitHub Project V2의 기본 `Status` 필드:
- **Todo** (기본)
- **In Progress**
- **Done**

추가 권장 옵션 (Settings → Status field 편집):
- **Blocked** — 선행 의존성 미해결
- **In Review** — PR 생성·머지 대기
- **Backlog** — 우선순위 낮음

## 6. 일정 변경 시 (재조정)

작업이 지연되거나 앞당겨질 때:
- **개별 이슈**: 이슈 카드 클릭 → 우측 패널의 `Start Date` / `End Date` 직접 수정
- **일괄 변경**: gh CLI 또는 스크립트 재실행 (`tasks/3.2_AI_Accelerated_Schedule.md` §4 매핑 표 수정 후 `scripts/setup-github-project.ps1` 재실행 — idempotent하게 Date만 갱신)

## 7. 알림 줄이기 (선택)

212개 이슈 생성 후 메일 폭탄을 줄이려면:
1. GitHub → Settings → Notifications → "Email" 섹션

## 9. 자주 묻는 질문

**Q. 이슈가 Roadmap에 안 보여요.**
→ Start Date 또는 End Date 중 하나가 비어 있음. 이슈 카드 열어서 Date 입력.

**Q. Group by에 "phase" 라벨이 안 보여요.**
→ Group by → Labels 선택 후 라벨 이름으로 자동 그룹화됨. 검색창에 `phase` 입력하면 필터됨.

**Q. 가로 막대 색이 안 구분돼요.**
→ Labels 색이 자동 적용. epic:* 라벨로 group by 하면 Epic별 색 분리.

**Q. 의존성 화살표를 보고 싶어요.**
→ GitHub Project Roadmap은 의존성 그래프 미지원. [`tasks/3.1_Gantt_Chart_Critical_Path.md`](3.1_Gantt_Chart_Critical_Path.md)의 Mermaid 다이어그램 참조.

**Q. 마일스톤 기능과 차이?**
→ Mileston = 이슈 그룹핑 (날짜 마감 1개). Project = 시각화 + 자동화 + 다중 필드. 본 프로젝트는 Project 위주, Milestone은 PoC/Phase 2 같은 큰 단위만.
