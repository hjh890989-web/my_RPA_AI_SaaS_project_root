### :dart: Summary
- **기능명**: [NFR-PERF-001] STT p95 ≤5,000ms 성능 테스트 스크립트 (2건/분 Throttle 적용)
- **목적**: 패시브 로깅(E1)의 핵심 기능인 STT(Speech-to-Text) 음성→텍스트 변환의 응답 시간이 **p95 기준 ≤5,000ms**를 달성하는지 검증하는 자동화된 성능 테스트 스크립트를 구현한다. Gemini API Free Tier의 Rate Limit(15 RPM)을 고려하여 2건/분 Throttle을 적용한 현실적 부하에서 측정한다.

### :white_check_mark: Task Breakdown (실행 계획)
- [ ] 성능 테스트 스크립트 작성: STT Server Action 호출 → 응답 시간 기록 반복 실행
- [ ] 테스트 오디오 픽스처 준비: 5초/10초/30초 길이의 한국어 음성 샘플 3종
- [ ] 2건/분 Throttle 적용: Rate Limiter 환경에서 실제 큐 대기 시간 포함 측정
- [ ] p95 통계 산출: 최소 30회 반복 실행 → p50/p95/p99/max 통계 계산

### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 5초 음성 → STT 변환 p95 ≤5,000ms**
- **Given**: 5초 길이의 한국어 음성 파일이 준비되어 있음
- **When**: STT Server Action을 30회 반복 호출함 (2건/분 Throttle 적용)
- **Then**: p95 응답 시간이 ≤5,000ms이며, 성능 리포트가 생성된다.

**Scenario 2: 30초 음성(최대 길이) → p95 ≤5,000ms 유지**
- **Given**: 30초 길이의 한국어 음성 파일
- **When**: 30회 반복 호출함
- **Then**: 음성 길이 증가에도 p95 ≤5,000ms가 유지된다.

**Scenario 3: p95 초과 시 테스트 FAIL**
- **Given**: STT 응답이 일시적으로 느려져 p95가 6,000ms를 기록함
- **When**: 성능 테스트가 완료되고 통계를 산출함
- **Then**: 테스트가 FAIL로 표시되고, `WARNING: STT p95 (6,000ms) exceeds threshold (5,000ms)` 경고가 출력된다.

### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 용량 사용량 정확 조회**
- **Given**: Supabase DB에 300MB/500MB(60%) 데이터가 적재됨
- **When**: 용량 모니터링 API를 호출함
- **Then**: `db_usage_mb=300`, `db_limit_mb=500`, `db_usage_pct=60%`, `storage_usage_mb`, `storage_limit_mb=1024` 정확히 반환.

**Scenario 2: 80% 도달 → 사전 경고 알림**
- **Given**: DB 사용량이 400MB/500MB(80%)에 도달함
- **When**: 일일 모니터링 체크가 실행됨
- **Then**: ADMIN + CIO에게 "DB 용량 80% 도달 (400MB/500MB). 데이터 정리 또는 유료 플랜 전환을 검토하세요." 경고 발송.

**Scenario 3: 데이터 증가 속도 → 고갈 예상일 산출**
- **Given**: 최근 30일간 일평균 2MB 증가, 현재 400MB/500MB
- **When**: 고갈 예상일을 산출함
- **Then**: "현재 증가 속도 기준 약 50일 후(2026-06-08) Free Tier 고갈 예상" 정보가 대시보드에 표시됨.

**Scenario 4: 90% 도달 → 긴급 알림**
- **Given**: DB 사용량이 450MB/500MB(90%)
- **When**: 모니터링 체크가 실행됨
- **Then**: CRITICAL 등급 긴급 알림 발송: "DB 용량 90% — 서비스 중단 위험. 즉시 조치 필요."
