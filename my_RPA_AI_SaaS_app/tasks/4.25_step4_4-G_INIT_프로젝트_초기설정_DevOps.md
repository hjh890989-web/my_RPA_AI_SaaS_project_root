### :dart: Summary
- **기능명**: [INIT-002] Vercel Free 배포 설정 + Git Push 자동 배포 파이프라인 구성
- **목적**: GitHub 리포지토리에 Push하면 **자동으로 Vercel에 배포**되는 CI/CD 파이프라인을 구성한다. Vercel Hobby (무료) 플랜을 사용하며, main 브랜치 Push = Production 배포, PR = Preview 배포를 설정한다.

- 제약 조건: CON-04 (Vercel Hobby Free), REQ-FUNC-030 (자동 배포)
- 선행 태스크: INIT-001

### :white_check_mark: Task Breakdown (실행 계획)
- [ ] Vercel 프로젝트 생성: GitHub 리포지토리 연결
- [ ] 빌드 설정: Framework Preset = Next.js, Root Directory = `./`
- [ ] 환경변수 설정: Vercel Dashboard에서 프로덕션 환경변수 등록
- [ ] main 브랜치 → Production 배포: Push 시 자동 배포
- [ ] PR → Preview 배포: PR 생성 시 Preview URL 자동 생성
- [ ] 배포 성공/실패 알림: GitHub Commit Status + Vercel 알림
- [ ] 커스텀 도메인 설정 (선택): `factoryai.vercel.app` 기본 제공

### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: main Push → 자동 Production 배포**
- **Given**: main 브랜치에 코드가 커밋됨
- **When**: `git push origin main` 실행
- **Then**: Vercel이 자동으로 빌드 + 배포를 수행하고, Production URL에서 최신 코드가 서빙됨.

**Scenario 2: PR → Preview 배포**
- **Given**: feature 브랜치에서 PR을 생성함
- **When**: PR이 GitHub에 등록됨
- **Then**: Vercel Preview URL이 자동 생성되고, PR 코멘트에 Preview 링크가 표시됨.

**Scenario 3: 빌드 실패 → 배포 차단**
- **Given**: 빌드 에러가 있는 코드를 Push함
- **When**: Vercel 빌드가 실행됨
- **Then**: Production 배포가 차단되고, 이전 성공 배포가 유지됨.

### :gear: Technical & Non-Functional Constraints
- **플랜**: Vercel Hobby (무료)
- **제한**: 빌드 시간 ≤45초 (Hobby 제한), 실행 시간 ≤10초 (Serverless)
- **브랜치 전략**: main = Production, feature/* = Preview
- **테스트**: 배포 후 `curl` 응답 코드 200 확인

### :checkered_flag: Definition of Done (DoD)
- [ ] main Push → Production 자동 배포가 동작하는가?
- [ ] PR → Preview 배포가 동작하는가?
- [ ] 환경변수가 Vercel Dashboard에 안전하게 등록되었는가?
- [ ] 빌드 실패 시 기존 배포가 유지되는가?
