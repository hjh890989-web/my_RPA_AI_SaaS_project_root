### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 정상 음성 녹음 → 텍스트 변환 성공**
- **Given**: 작업자(OPERATOR)가 명확한 한국어 음성("금속 절삭 완료, 표면 상태 양호")을 10초간 녹음함
- **When**: `POST /api/v1/log-entries`로 음성 데이터를 전송함
- **Then**: Gemini STT가 텍스트로 변환하고, 변환 정확도 ≥85%이며, LOG_ENTRY에 `status=PENDING`으로 저장되고, 전체 처리 p95 ≤5초를 만족한다.

**Scenario 2: 소음 환경 음성 녹음 → 변환 + 품질 경고**
- **Given**: 공장 소음이 있는 환경에서 녹음된 음성 데이터가 주어짐
- **When**: STT 변환을 요청함
- **Then**: 변환은 수행되되, Confidence Score가 임계치 이하인 경우 `quality_warning` 플래그가 LOG_ENTRY에 함께 기록된다.

**Scenario 3: Rate Limit 도달 시 큐잉 처리**
- **Given**: Gemini API Rate Limit(15 RPM)에 도달한 상태
- **When**: 추가 STT 변환 요청이 발생함
- **Then**: 요청이 In-memory Queue에 적재되고, 202 Accepted를 반환하며, Exponential Backoff 후 순차 처리된다.

**Scenario 4: 미인가 사용자 접근 차단**
- **Given**: VIEWER 역할의 사용자가 로그인되어 있음
- **When**: `POST /api/v1/log-entries`로 음성 데이터를 전송함
- **Then**: 403 Forbidden이 반환되고, CISO에게 미인가 접근 알림이 ≤10초 내에 발송된다.

### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 동시 5건 녹음 → 전건 순차 처리 완료**
- **Given**: 5명의 작업자가 동시에 녹음 완료 후 STT 변환을 요청함
- **When**: 5건의 `POST /api/v1/log-entries` 요청이 동시에 도착함
- **Then**: 모든 5건이 In-memory Queue에 적재되고, FIFO 순서로 순차 처리되며, 전건 처리 완료까지 ≤15초 이내에 완료된다. **드롭된 요청 = 0건**.

**Scenario 2: 동시 10건 녹음 → Queue 대기 + 완료**
- **Given**: 10건의 STT 변환 요청이 동시 도착 (15 RPM 한도 내)
- **When**: Rate Limiter가 Gemini API 호출을 제어함
- **Then**: 모든 10건이 큐에 적재되고 순차 처리된다. 드롭 0건이며, 사용자에게 "처리 중" 진행률이 표시된다.

**Scenario 3: Rate Limit 초과 시 Exponential Backoff**
- **Given**: 15 RPM 한도를 초과하는 요청 속도
- **When**: Gemini API가 429 Too Many Requests를 반환함
- **Then**: Exponential Backoff(1초→2초→4초)로 재시도하며, 최종적으로 전건 처리 완료된다. 드롭 0건.

**Scenario 4: 극단적 대량 요청 → Graceful Degradation**
- **Given**: 50건의 동시 STT 요청이 발생 (비정상 상황)
- **When**: 큐 용량이 극단적으로 증가함
- **Then**: 큐 최대 크기(configurable)를 초과하는 요청에 대해 503 Service Unavailable을 반환하되, 이미 큐에 적재된 요청은 정상 처리된다.
