### :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: PENDING 리포트 다운로드 시도 → ≤1초 차단 + 알림 ≤10초**
- **Given**: `status=PENDING`인 AUDIT_REPORT(id=RPT-001)가 존재함
- **When**: 사용자가 RPT-001의 PDF 다운로드를 시도함
- **Then**: ≤1초 내에 403 Forbidden이 반환되고, `message: "미승인 리포트는 다운로드할 수 없습니다. 품질이사의 승인 후 이용해 주세요."`, ADMIN에게 ≤10초 내 "미승인 발행 시도" 알림이 발송된다.

**Scenario 2: APPROVAL 레코드 없음 → Fail-Close 차단**
- **Given**: 특정 리소스에 대한 APPROVAL 레코드가 아예 존재하지 않음
- **When**: 해당 리소스의 외부 발행을 시도함
- **Then**: Fail-Close 원칙에 따라 즉시 차단(403). "승인 기록 없음" 사유가 AUDIT_LOG에 기록됨.

**Scenario 3: APPROVED 상태 → 정상 발행 허용**
- **Given**: `status=APPROVED`인 AUDIT_REPORT가 존재하고, 승인자 정보가 기록됨
- **When**: PDF 다운로드를 요청함
- **Then**: 정상적으로 PDF가 반환되고, AUDIT_LOG에 "승인 후 발행" 이벤트가 기록됨.

**Scenario 4: 직접 URL 우회 시도 → 차단**
- **Given**: PENDING 리포트의 직접 Storage URL을 알고 있음
- **When**: 브라우저에서 직접 URL로 접근함
- **Then**: 인증/인가 미들웨어에서 차단되고, CISO에게 "우회 시도" 긴급 알림이 발송됨.

**Scenario 1: 헬스체크 3회 실패 → ≤5분 내 수동 판단 모드 전환**

- **Then**: ≤5분(3분째 실패 시점 + 2분 이내) 내에 시스템이 `MANUAL_FALLBACK` 모드로 전환되고, 모든 이상 징후 알림에 XAI 설명 대신 원본 데이터가 포함되어 인간 판단자에게 전달된다.

- **Given**: `MANUAL_FALLBACK` 모드가 활성화된 상태
- **When**: 이상 징후(온도 초과)가 감지됨

- **When**: 30분간 연속 성공이 확인됨
- **Then**: 시스템이 정상 모드(XAI 활성)로 자동 복귀하고, 이후 이상 징후에 대해 XAI 설명이 다시 생성된다.

**Scenario 4: 복구 후 15분 만에 재실패 → 모드 유지**
- **Given**: 복구 후 15분 시점에 헬스체크가 다시 실패함 (30분 미도달)
- **When**: 안정성 관찰 기간 내 재실패 감지
- **Then**: `MANUAL_FALLBACK` 모드가 유지되고, 30분 카운터가 리셋된다.

**Scenario 5: 헬스체크 2회만 실패 → 모드 전환 없음 (대조군)**
