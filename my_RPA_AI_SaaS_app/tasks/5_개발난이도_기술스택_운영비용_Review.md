# FactoryAI — 개발 난이도, 기술 스택, 운영 비용 검토 보고서

본 보고서는 Step 1~4 통합 명세서를 바탕으로 **① 개발 난이도(MVP 적절성)**, **② 기술 스택(복잡성 및 오픈소스 여부)**, **③ 운영 소요 비용(클라우드 기반)** 세 가지 관점에서 심층 검토한 결과입니다.

---

### 1. 개발 난이도 및 MVP 단계 적절성 평가

전체적으로 보안과 데이터 정합성에는 높은 비중을 두면서도, 인프라 관리는 클라우드 서비스에 전적으로 위임하여 비즈니스 로직에만 집중할 수 있도록 구성되어 있습니다.

| 단계 (Step) | 주요 구현 영역 | 개발 난이도 | MVP 적절성 평가 및 사유 |
| :--- | :--- | :---: | :--- |
| **Step 1. 데이터/계약** | DB/ORM 설계, API 규격 설계 | **보통** | MVP로서는 엔드포인트가 다소 많으나, Zod와 Prisma를 통한 엄격한 타입 제어는 B2B 데이터 무결성의 기초이므로 적절함. |
| **Step 2. 로직 구현** | AI 연동(STT/Vision/XAI), ERP 연동 (3중 차단) | **높음** | 외부 LLM API(Gemini) 사용으로 난제를 해결함. ERP "비파괴(Read-Only) 동기화" 채택으로 MVP 리스크를 획기적으로 낮춤. |
| **Step 3. 테스트** | RBAC/인가 시나리오, E2E 통합 테스트 | **보통~높음** | 방대한 테스트 요구사항이 있으나, "보안과 감사 가능성"이 코어 밸류인 제품이므로 타협할 수 없는 적절한 스펙임. |
| **Step 4. 비기능/인프라** | Serverless CI/CD, 모니터링 알림 | **보통** | Vercel/Supabase 매니지드 서비스를 100% 활용해 장애 허용과 배포 난이도를 낮춘 최적의 접근법. |

---

### 2. 기술 스택 분석 (복잡성 및 오픈소스 현황)

풀스택 TypeScript 단일 생태계로 진입 장벽을 최소화했으며, 벤더 락인 방지를 위한 추상화 계층을 적절히 사용했습니다.

| 레이어 (Layer) | 주요 채택 기술 스택 | 시스템 복잡성 | 오픈소스 / 상용 여부 | 도입 이점 |
| :--- | :--- | :---: | :---: | :--- |
| **Web Framework**| Next.js 15, TypeScript | **보통** | 오픈소스 (MIT) | 단일 풀스택으로 개발 생산성 및 타입 안정성 확보 |
| **UI Component** | Tailwind CSS, shadcn/ui, Recharts | **낮음** | 오픈소스 (MIT) | 미려한 B2B 대시보드를 빠르게 프로토타이핑 가능 |
| **DB & ORM** | Prisma, Supabase (Postgres) | **보통** | 오픈소스 기반 | 로컬(SQLite)과 클라우드 환경의 유연한 스왑 지원 |
| **AI Abstraction** | Vercel AI SDK, Gemini API | **보통** | 오픈소스 / 상용 API | 벤더 중립적 구조로 향후 모델 교체가 용이함 |
| **PDF Generation** | @react-pdf/renderer | **낮음** | 오픈소스 (MIT) | 규제 리포트 생성을 클라이언트 단에서 처리하여 서버 부하 방어 |
| **Security/Auth** | NextAuth v5, Jest, Playwright | **보통** | 오픈소스 (MIT) | 엔터프라이즈급 인증 및 BDD 기반 행동 제어 검증 |

### 3. 클라우드 기반 운영 소요 비용 (월간 예비 산정)

Step 4의 제어 로직을 기반으로 산정했을 때, B2B PoC 단계까지 **월 고정비 $0(무료)** 운영이 가능하도록 최적화되어 있습니다.

| 클라우드 서비스 | 사용 목적 | 권장 도입 플랜 | 예상 월별 비용 | 비용 최적화 포인트 |
| :--- | :--- | :--- | :---: | :--- |
| **Vercel** | 호스팅, Serverless API 구동 | **Free (Hobby)** | **$0** | 클라이언트 사이드 프로세싱으로 서버리스 함수 실행 시간 절약 |
| **Supabase** | Postgres DB, 보안망(RLS) | **Free Tier** | **$0** | Step 4의 용량 모니터링을 통해 500MB 한도 내 효율적 관리 |
| **Google Gemini** | AI 파싱 및 설명 생성 | **Free Tier** | **$0** | **Rate Limiter(AI-002) 아키텍처**를 통해 15 RPM 한도 내 무과금 유지 |
| **GitHub** | 소스 관리, CI/CD 연동 | **Free** | **$0** | 무상 보안 소스 관리 및 파이프라인 연동 |
| **총계 (Monthly)**| | **초기 MVP** | **총 $0** | **고객사 확대 전까지 자본 투입 완전 차단 가능** |

### 💡 총평
본 설계는 기술적인 복잡도는 엔터프라이즈 급(B2B 보안/감사 지원)을 지향하면서도, 운영 인프라는 클라우드 Free Tier 생태계를 철저히 활용하도록 우회 설계되었습니다. 특히 **Rate Limiting 처리와 클라이언트 사이드 PDF 생성** 등은 인프라 비용 부담을 사용자/모델 측으로 분산시킨 전략적 선택으로 평가됩니다.
